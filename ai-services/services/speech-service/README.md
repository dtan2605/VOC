# VOC Speech Service v2.0.0

AI-powered IELTS speaking practice service with real-time conversation, speech-to-text, text-to-speech, and intelligent scoring.

## Architecture

```
speech-service/
├── app/
│   ├── main.py                 # FastAPI application entry point
│   ├── config.py               # Configuration management
│   ├── database.py             # Connection pooling & schema
│   ├── models/                 # Pydantic schemas & dataclasses
│   ├── services/               # Business logic (LLM, STT, TTS, Memory, Scoring)
│   ├── repositories/           # Database operations
│   ├── routes/                 # API endpoints (REST & WebSocket)
│   └── utils/                  # Helper functions
├── tests/                      # Unit tests
├── Dockerfile
├── requirements.txt
└── .env.example
```

## Key Features

### 1. **LLM-Powered Responses** (Groq/Ollama)
- All responses generated by LLM (no hardcoded NLP)
- Groq as primary provider (llama3-8b-8192)
- Ollama as fallback
- Automatic retry with exponential backoff
- Rate limit handling

### 2. **Speech-to-Text** (Faster-Whisper)
- Real-time audio transcription
- VAD (Voice Activity Detection) filtering
- Multiple language support

### 3. **Text-to-Speech** (Kokoro)
- High-quality neural TTS
- Configurable voice and speed
- WAV output format

### 4. **Intelligent Scoring**
- IELTS-style scoring (1-9 scale)
- Four criteria: Fluency, Lexical Resource, Grammar, Pronunciation
- Error detection with corrections
- Rephrasing suggestions

### 5. **Conversation Memory**
- LLM-based memory extraction
- Persistent across turns
- Automatic deduplication

### 6. **Real-time WebSocket**
- Low-latency audio streaming
- Bidirectional communication
- Automatic reconnection

## API Endpoints

### REST API

```
GET  /health                           # Health check
GET  /api/speaking/health              # Health check (alternative)
POST /api/speaking/sessions            # Create new session
GET  /api/speaking/sessions/{id}       # Get session details
POST /api/speaking/sessions/{id}/turns # Submit turn (text/audio)
```

### WebSocket

```
WS /api/speaking/ws
```

**Message Types:**
- `start` - Initialize session
- `audio_start` / `audio_end` - Audio streaming
- `text` - Text input
- `reset` - Reset session

## Configuration

### Environment Variables

```bash
# Database
SPEECH_DB_HOST=mysql
SPEECH_DB_PORT=3306
SPEECH_DB_NAME=voc_speaking
SPEECH_DB_USER=voc_user
SPEECH_DB_PASSWORD=<your-password>
SPEECH_DB_POOL_SIZE=10

# Groq (Primary LLM)
GROQ_API_KEY=<your-groq-api-key>
GROQ_MODEL=llama3-8b-8192
GROQ_TIMEOUT=20.0
GROQ_MAX_RETRIES=2

# Ollama (Fallback LLM)
OLLAMA_BASE_URL=http://ollama:11434
OLLAMA_MODEL=llama3.2:1b

# Whisper STT
WHISPER_MODEL=small.en
WHISPER_DEVICE=cpu
WHISPER_COMPUTE_TYPE=int8

# Kokoro TTS
KOKORO_MODEL_PATH=/app/models/tts/kokoro-v1.0.onnx
KOKORO_VOICES_PATH=/app/models/tts/voices-v1.0.bin
TTS_VOICE=af_bella
TTS_SPEED=1.0

# Rate Limiting
RATE_LIMIT_PER_MINUTE=30
MAX_AUDIO_SIZE_MB=10
```

## Installation

### Docker (Recommended)

```bash
# Build image
docker build -t voc-speech-service .

# Run with environment
docker run -p 8003:8003 \
  --env-file .env \
  voc-speech-service
```

```bash

# Set environment variables
export GROQ_API_KEY=your_key_here
```

## Troubleshooting

### Groq API Issues
- Verify API key is set: `echo $GROQ_API_KEY`
- Check rate limits: https://console.groq.com/
- Test endpoint: `curl -H "Authorization: Bearer $GROQ_API_KEY" https://api.groq.com/openai/v1/models`

### Database Connection
- Verify MySQL is running: `docker ps | grep mysql`
- Check credentials in `.env`
- Test connection: `mysql -h $SPEECH_DB_HOST -u $SPEECH_DB_USER -p`

### Whisper/TTS Models
- Models are downloaded at container startup
- Check logs: `docker logs speech-service`
- Verify model paths in `.env`

### WebSocket Connection
- Ensure nginx proxy is configured for WebSocket upgrade
- Check browser console for connection errors
- Test with WebSocket client: `wscat -c ws://localhost:8003/api/speaking/ws`

## Performance Tuning

### Database
- Increase `SPEECH_DB_POOL_SIZE` for high concurrency
- Enable MySQL query cache
- Add indexes for frequently queried columns

### LLM
- Use faster Groq models for lower latency
- Enable Ollama caching
- Adjust `GROQ_TIMEOUT` based on network conditions

### Audio Processing
- Reduce `WHISPER_MODEL` size for faster STT (tiny.en vs small.en)
- Enable GPU acceleration: `WHISPER_DEVICE=cuda`
- Adjust `TTS_MAX_WORDS` to balance quality vs speed
